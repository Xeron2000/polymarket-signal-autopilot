# Real Data Quickstart (Polymarket REST + WS)

## 1) Build and verify

```bash
go mod tidy
go build ./...
go test ./...
```

## 1.5) Apply database migrations

Run all SQL migrations in order before starting worker:

```bash
psql "$DATABASE_URL" -f migrations/001_raw_events.sql
psql "$DATABASE_URL" -f migrations/002_news_events.sql
psql "$DATABASE_URL" -f migrations/003_linked_events.sql
psql "$DATABASE_URL" -f migrations/004_execution_pipeline.sql
psql "$DATABASE_URL" -f migrations/005_runtime_state.sql
```

## 2) Start API process

By default the API process binds `:8080` and queries official public endpoints.

```bash
API_ADDR=:8080 \
POLY_GAMMA_URL=https://gamma-api.polymarket.com \
POLY_CLOB_URL=https://clob.polymarket.com \
go run ./cmd/api
```

Available routes:

- `GET /health`
- `GET /markets?active=true`
- `GET /events`
- `GET /orderbook?token_id=<TOKEN_ID>`
- `GET /trades?token_id=<TOKEN_ID>&since=<RFC3339>`

## 3) Start worker process (persistent DB + signal engine)

The worker subscribes to real market WS feed, writes raw events to PostgreSQL,
detects rank-shift lead-lag signals, sends Telegram alerts, and can optionally
submit private signed orders.

Get token IDs first (official Gamma API returns market metadata with token ids):

```bash
curl -s "https://gamma-api.polymarket.com/markets?active=true&limit=5" | jq
```

Or use the built-in selector to auto-pick candidate token IDs by liquidity/volume rules:

```bash
go run ./cmd/assetpicker \
  --tier1-only \
  --core-categories "Sports,Soccer,Politics" \
  --core-top-pct 0.30 \
  --other-top-pct 0.10 \
  --max-selected-markets 12 \
  --max-asset-ids 24
```

The command prints ready-to-copy env lines: `POLY_ASSET_IDS`, `ARB_ASSET_A`, `ARB_ASSET_B`.

Then pick token IDs from `clobTokenIds` / `clob_token_ids` values and run worker:

```bash
DATABASE_URL=postgres://user:pass@localhost:5432/polysignal?sslmode=disable \
POLY_ASSET_IDS=<TOKEN_ID_1>,<TOKEN_ID_2> \
POLY_WS_URL=wss://ws-subscriptions-clob.polymarket.com/ws/market \
ARB_ASSET_A=<TOKEN_ID_1> \
ARB_ASSET_B=<TOKEN_ID_2> \
ARB_FLIP_SPREAD=0.05 \
ARB_COOLDOWN_SECONDS=300 \
TG_BOT_TOKEN=<BOT_TOKEN> \
TG_CHAT_ID=<CHAT_ID> \
WORKER_OPS_ADDR=:9091 \
go run ./cmd/worker
```

Required env:

- `DATABASE_URL` (PostgreSQL DSN)
- `POLY_ASSET_IDS` (comma-separated market token IDs)
- `ARB_ASSET_B` (or provide at least two `POLY_ASSET_IDS`)

Optional env:

- `POLY_WS_URL` (defaults to official market WS URL)
- `ARB_ASSET_A` (defaults first asset in `POLY_ASSET_IDS`)
- `ARB_ASSET_A_URL` / `ARB_ASSET_B_URL` (optional; override clickable market links in Telegram)
- `ARB_TARGET_LABEL` (used in notifications)
- `ARB_FLIP_SPREAD` (default `0.05`)
- `ARB_COOLDOWN_SECONDS` (default `300`)
- `TG_BOT_TOKEN` + `TG_CHAT_ID` (enable Telegram notifications)
- `SIGNAL_NOTIFY_COOLDOWN` (default `2m`, dedupe repeated signal notifications)
- `WORKER_OPS_ADDR` (default `:9091`, worker ops server)
- `OPS_ALERT_COOLDOWN` (default `2m`, dedupe for operational alerts)
- `ORDER_RECONCILE_MAX_ATTEMPTS` (default `3`)
- `ORDER_RECONCILE_BASE_DELAY` (default `300ms`)
- `ORDER_RECONCILE_MAX_DELAY` (default `2s`)
- `ORDER_RECONCILE_SWEEP_INTERVAL` (default `15s`, periodic re-scan for non-terminal orders)
- `ORDER_RECONCILE_SWEEP_LIMIT` (default `50`)
- `ANOMALY_SEVERITY_MEDIUM` (default `0.08`)
- `ANOMALY_SEVERITY_HIGH` (default `0.15`)
- `ANOMALY_ALERT_MIN_SEVERITY` (default `medium`, values: `low|medium|high`)

### Enable auto execution (private signed order endpoint)

Set `AUTO_EXECUTE=true` and provide L2 credentials. Recommended mode is dynamic order signing with `POLY_PRIVATE_KEY`.

```bash
AUTO_EXECUTE=true \
POLY_TRADER_ADDRESS=<0x_funder_or_profile_address> \
POLY_API_KEY=<api_key> \
POLY_PASSPHRASE=<passphrase> \
POLY_API_SECRET=<base64_secret> \
POLY_PRIVATE_KEY=<0x_eoa_private_key> \
POLY_ORDER_OWNER=<api_key_or_owner_uuid> \
POLY_ORDER_PRICE=0.50 \
POLY_ORDER_SIDE=BUY \
POLY_ORDER_TYPE=GTC \
POLY_ORDER_SIGNATURE_TYPE=0 \
POLY_ORDER_FEE_BPS=0 \
POLY_ORDER_NONCE=0 \
POLY_ORDER_EXPIRATION_SECONDS=600 \
POLY_CHAIN_ID=137 \
POLY_DYNAMIC_PRICING=true \
POLY_DYNAMIC_PRICE_OFFSET_BPS=25 \
POLY_DYNAMIC_PRICE_MIN=0.01 \
POLY_DYNAMIC_PRICE_MAX=0.99 \
POLY_DYNAMIC_PRICE_TIMEOUT_MS=1500 \
EXEC_COUNTRY=US \
EXEC_MARKET_CAP=1000 \
EXEC_DAILY_LOSS_CAP=500 \
EXEC_NOTIONAL=10 \
go run ./cmd/worker
```

Notes:

- Dynamic mode signs each order payload per signal and token ID.
- Dynamic nonce mode uses `runtime_state` for DB atomic nonce allocation (seeded by `POLY_ORDER_NONCE` on first allocation).
- Dynamic pricing mode derives a limit price from current top-of-book and falls back to `POLY_ORDER_PRICE` on lookup failure.
- Post-submit reconciliation backchecks order status (`GetOrder`) and appends reconciled status records.
- L2 signature headers (`POLY_*`) are generated by the worker for private endpoint calls.
- If `POLY_PRIVATE_KEY` is not set, worker falls back to legacy `POLY_SIGNED_ORDER_JSON` template mode.

Worker ops endpoints:

- `GET http://<worker-host>:9091/health` (when `WORKER_OPS_ADDR=:9091`)
- `GET http://<worker-host>:9091/metrics` (when `WORKER_OPS_ADDR=:9091`)
- `GET http://<worker-host>:9091/anomaly-panel?limit=20` (JSON panel)
- `GET http://<worker-host>:9091/anomaly-panel.html?limit=20` (HTML panel)
